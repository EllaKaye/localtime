localtime extension example

The localtime shortcode displays a given date and time in the reader’s local timezone. The “Renders as” column below updates to your local time when the page loads.

No-padding format tokens

Use %- variants to suppress zero-padding on numeric fields.

Shortcode Renders locally as
{{< localtime 2026-01-03 08:05 UTC format="%d %B %Y at %H:%M" >}}

2026-01-03 08:05 UTC
{{< localtime 2026-01-03 08:05 UTC format="%-d %B %Y at %-H:%M" >}}

2026-01-03 08:05 UTC
{{< localtime 2026-01-03 08:05 UTC format="%-d %b %Y, %-I:%M %p" >}}

2026-01-03 08:05 UTC

The first row uses %d and %H (zero-padded), the second uses %-d and %-H (no padding).

IANA timezone names

Any IANA timezone name can be used directly. Unlike abbreviations, IANA names are unambiguous and the browser handles DST automatically.

Shortcode Renders locally as
{{< localtime 2026-01-30 13:00 America/New_York >}}

2026-01-30 13:00 America/New_York
{{< localtime 2026-07-15 13:00 America/New_York >}}

2026-07-15 13:00 America/New_York
{{< localtime 2026-01-30 09:00 Europe/Paris >}}

2026-01-30 09:00 Europe/Paris
{{< localtime 2026-07-15 09:00 Europe/Paris >}}

2026-07-15 09:00 Europe/Paris
{{< localtime 2026-01-30 10:00 Australia/Sydney >}}

2026-01-30 10:00 Australia/Sydney

The January and July rows for the same timezone show different UTC offsets because the browser applies the correct DST offset for each date.

Supported timezone formats

  • Common abbreviations: UTC, GMT, EST, EDT, CST, CDT, MST, MDT, PST, PDT, CET, CEST, BST, JST, AEST, AEDT, NZST, NZDT, and more
  • IANA timezone names: America/New_York, Europe/Paris, Asia/Tokyo, etc.
  • UTC offsets: +05:30, -08:00, +0530, -8
  • Prefixed offsets: UTC+5, UTC-8, GMT+1

DST handling

DST is handled automatically in the reader’s browser. Common abbreviations that observe DST (e.g. EST, CET) are mapped to their IANA timezone, and the browser applies the correct offset for each specific date.

Shortcode Renders locally as
{{< localtime 2026-07-15 14:00 CET >}}

2026-07-15 14:00 CET
{{< localtime 2026-01-15 14:00 CET >}}

2026-01-15 14:00 CET
{{< localtime 2026-07-15 09:00 EST >}}

2026-07-15 09:00 EST
{{< localtime 2026-01-15 09:00 EST >}}

2026-01-15 09:00 EST

The July and January rows for the same abbreviation use different offsets (e.g. CEST vs CET, EDT vs EST) because the browser’s timezone database applies the correct rule for each date.
